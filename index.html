Well, hello there!

My name is Andrew Palmer. I am a full-stack developer based in Cape Girardeau, Missouri. I certainly didn't take the traditional career route to web development; while I did some programming in high school and wrote HTML in the dial-up days of the web, I never dreamed of doing so professionally until just a few years ago.

In 2015, I left my job managing a convenience store to take a tech support position for a Real Estate Marketing company. Within six months, I had been promoted to a role that involved some simple PHP scripting and database management, and in late 2017, I earned a spot on the development team developing new products. I am always on the lookout for new challeneges, and haven't met a language or library yet that scared me.

TicketTracker.

  • Final project for MTI's Code Labs One
  • Rails Web App with PostGreS replaced by MySQL
  • Coudinary Integration for use with containerized deployment
  • Demo running on Heroku, login as guest@guest.com , pass 'Test123'

This was a fun one. As a graduation requirement of the Code Labs One bootcamp, we had to make a web app of our choosing, so I settled on making a simple ticket system. After about a month of work, I had an anachronisticly styled, MySQL-driven Rails app that is surprisingly customizable for use in tech support, development, or really anywhere that you might use a Trello board. Tickets support adding screenshots, can be commented on by other users (but only edited by the original user or Admin), and are automatically archived when the ticket is marked as 'Closed.' My favorite part of this project besides Rails itself, which is FABULOUS for rapid prototyping, was the fact that as I progressed, I was actually able to use TicketTracker to document it's own development.
